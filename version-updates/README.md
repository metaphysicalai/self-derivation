## 버전 규칙

| 유형 | 표기 | 기준 | 예시 |
|:---|:---|:---|:---|
| **Major** | X.0 | 제목 변경 | 1.0 → 2.0 |
| **Minor** | X.Y | 피어리뷰 반영 | 1.1 → 1.2 |
| **Patch** | X.Y.Z | 자체 수정 | 1.2 → 1.2.1 |

## 파일 구조

```
version-updates/
├── README.md          # 변경 이력 요약
└── log/               # 상세 기록
    ├── 1.1-1.2.md
    ├── 1.2-1.2.1.md
    └── ...
```

각 버전 업데이트는 `log/[이전버전]-[새버전].md` 형식으로 기록한다.

- **포함 내용**: 변경 사항, 수정 근거 (Minor는 리뷰어 지적사항 포함)
- **목적**: 수정 근거 추적, 유사 지적 재발 시 대응 자료
